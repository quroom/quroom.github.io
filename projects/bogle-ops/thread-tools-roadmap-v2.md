# Thread Tools Roadmap v2 (AI 위임 실행형)

목표: Threads 콘텐츠를 **기획 → 제작 → 검수 → 아카이브**까지 AI에게 안정적으로 위임 가능한 운영체계로 만든다.

---

## 0) 운영 원칙 (항상 고정)
- 철학: 존 보글 철학(저비용, 광범위 분산, 장기, 규칙)
- 톤: 실전형 + 공감형 + 과장 금지
- 감정 키워드: FOMO, 지루함, 남의 떡 심리 다루기
- 마무리: 커뮤니티 참여 질문(CTA)
- 금지: 투자 권유/확정적 수익 표현/과도한 선동 문구

---

## 1) AI 위임 구조 (Agent 단위)

### Agent A. Topic Planner (주제 기획)
**입력**
- `cafe-topic-bank.md`
- 최근 7일 게시 로그
- 금일 캠페인 목표(예: 댓글 유도/저장 유도)

**출력**
- 오늘 주제 3개 (우선순위 포함)
- 각 주제별 훅 3개
- 중복 위험 체크

**완료 기준 (DoD)**
- 전일/최근 7일 주제와 중복 없음
- 초보자 관점 문장 포함
- 각 주제에 CTA 방향 1개 명시

---

### Agent B. Draft Writer (초안 작성)
**입력**
- 선택 주제 1개
- 톤앤매너 규칙
- 포맷 규칙(`---`, 문단 길이, 개행)

**출력**
- Threads 본문 1개 (짧은형)
- 대체 버전 2개 (순한형/날카로운형)

**완료 기준 (DoD)**
- 앞/뒤 문구 규칙 준수
- CTA 포함
- 과장/단정 문장 제거

---

### Agent C. Humor Pack Generator (대량 문구 생성)
**입력**
- 주제군(예: 액티브 vs 인덱스, 멘탈 관리)
- 목표 개수(예: 100)
- 반복 문구 규칙(앞문구/뒷문구)

**출력**
- `.md` 팩 파일 1개
- 글감 사이 `---` 정렬 완료

**완료 기준 (DoD)**
- 중복 최소화(유사 문장 제거)
- 각 글감 독립성 확보
- md-viewer에서 개행 가독성 검증

---

### Agent D. QA Editor (품질/정합성 검수)
**입력**
- 작성된 `.md`/`.html`

**출력**
- 수정 포인트 체크리스트
- 최종 승인/보류 판단

**완료 기준 (DoD)**
- 링크 깨짐 없음
- 모바일 가독성 통과
- 캐시 이슈 대응(rev/query/version) 반영

---

### Agent E. Publisher Ops (배포/버전 관리)
**입력**
- 최종 파일
- 배포 경로

**출력**
- 버전 파일 생성(vN)
- 링크 업데이트(index/internal docs)
- 커밋/푸시 리포트

**완료 기준 (DoD)**
- remote push 완료
- md-viewer 링크 확인 가능
- 필요 시 alias(v2/v4 등) 유지

## 2) 바로 실행 가능한 위임 프롬프트 템플릿

### 템플릿 A: 오늘 주제 선정
```text
너는 Topic Planner다.
목표: 오늘 Threads용 주제 3개 선정.
조건:
- 존 보글 철학 기반
- 초보자 공감형
- 최근 주제와 중복 금지
출력:
1) 주제명
2) 왜 지금 이 주제인지(1문장)
3) 훅 3개
4) CTA 2개
```

### 템플릿 B: 1개 글 초안 생성
```text
너는 Draft Writer다.
입력 주제: <주제명>
톤: 실전형 + 공감형 + 과장 금지
형식:
- 짧은 문단
- 마지막 CTA
- 투자 권유/수익 단정 금지
출력:
- 본문 1개
- 대체안 2개(순한형/강한형)
```

### 템플릿 C: 유머팩 100개 생성
```text
너는 Humor Pack Generator다.
주제: <주제군>
개수: 100
규칙:
- 각 글 독립
- 글 사이 '---'
- 중복 표현 최소화
- 존 보글 철학 회귀 포인트 포함
출력: 마크다운 원문만
```

### 템플릿 D: 최종 QA
```text
너는 QA Editor다.
아래 문서를 검수하고 수정 지시를 10개 이내로 제시해라.
점검 항목:
- 철학 정합성
- 과장/단정 표현
- 가독성(모바일 포함)
- 링크/버전/rev
결론: 승인 or 보류
```
