# PRD v2 — Interest-based Bilingual Story English Service

## 0) 문서 정보
- 상태: Draft v2
- 대상: 초등 영어 입문~초중급 (특히 파닉스 미완성 학습자)
- 목표: "재미있는 이야기"를 통해 영어 학습 지속률을 높이고, 읽기/듣기/말하기 기초를 안정적으로 형성

---

## 1) Problem (문제 정의)

### 1-1. 사용자 문제
1. 아이가 영어를 "시험 과목"으로만 느껴 학습 지속이 어려움
2. 관심사와 영어 학습 콘텐츠가 연결되지 않아 몰입이 낮음
3. 난이도 미스매치(특히 파닉스 미완성 단계에서 문장 독해를 강요)
4. 학습 후 출력(의견 말하기/쓰기)이 약해 학습 효과가 누적되지 않음

### 1-2. 부모 문제
1. 어떤 수준 콘텐츠를 줘야 하는지 판단이 어려움
2. 비교 불안/거부감이 생겼을 때 즉시 조정 기준이 없음
3. 학습은 했는데 "무엇이 늘었는지" 확인이 어려움

---

## 2) Vision (제품 비전)
"아이 관심 주제"를 입력하면, 같은 내용을
- 그림 + 한글 버전
- 그림 + 쉬운 영어 버전
으로 제공하고,
챕터마다 클릭형 질문으로 다음 이야기를 분기해 **영어를 계속 하게 만드는 학습 경험**을 만든다.

---

## 3) Target User (타깃 사용자)

### 3-1. Primary
- 초4~초6
- 영어 입문~초급
- 특히 파닉스 미완성(소리-문자 매핑이 불안정한 단계)

### 3-2. Secondary
- 학부모(학습 관리자)
- 초등 영어 교사/튜터(보조 운영)

---

## 4) 핵심 가치 제안
1. **흥미 기반 몰입**: 아이 관심 주제로 시작
2. **이해 가능한 입력**: 한글/쉬운영어 병렬 제공
3. **능동형 학습**: 질문 선택으로 이야기 진행
4. **출력 강화**: 의견 남기기(한글→영어 확장)
5. **레벨 안전장치**: 파닉스 미완성 모드 제공

---

## 5) Level & Learning Mode (레벨/학습 모드)

## 5-1. 필수 기능: "초입문(파닉스 미완성) 모드"
사용자가 프로필에서 아래를 설정할 수 있어야 함:

- 영어 레벨
  - L0: 파닉스 입문(문장 최소)
  - L0.5: **I Can Read My First/Biscuit 수준** (Pre-A1~A1 초반, 반복 문장 중심)
  - L1: 파닉스 진행(짧은 문장)
  - L2: 기초 독해(A1)
  - L3: 확장 독해(A1~A2)

- 학습 모드
  - 읽기 중심
  - 듣기+따라말하기 중심
  - 질문응답 중심

- 문장 난이도 옵션
  - 문장 길이: 4~8 / 6~10 / 8~12 단어
  - 새 단어 수: 챕터당 3 / 5 / 8개
  - 반복 노출 횟수: 낮음/보통/높음

## 5-2. 파닉스 미완성 모드 동작
- 단어 하이라이트(소리 단위)
- 문장 TTS 느린 속도 재생
- "한 문장씩" 보기
- 선택형 질문 우선(서술형 최소)
- 한글 힌트 항상 표시 옵션

---

## 6) MVP Scope (1차 범위)

### 6-1. 포함
1. 관심 주제 입력 (예: 역사, 대체역사, 과학)
2. 챕터 1~3 생성
3. 한글/쉬운영어 버전 동시 제공
4. 챕터당 질문 카드 3개
5. 클릭 선택 시 다음 분기 생성(1단계)
6. 의견 기록(한글/영어)
7. 부모용 주간 리포트(학습일/읽은 챕터/단어/스트레스)
8. 레벨 설정(초입문 모드 포함)

### 6-2. 제외
- 실시간 다자간 협업
- 고급 문법 자동 교정
- 교사 LMS 완전 통합

---

## 7) User Flow
1. 부모/아이 프로필 생성
2. 레벨 설정(초입문 모드 선택 가능)
3. 관심 주제 선택
4. 챕터 1 읽기(한글→영어)
5. 질문 카드 클릭(분기)
6. 의견 입력
7. 다음 챕터 이어서 진행
8. 주간 리포트 확인

---

## 8) 콘텐츠 정책 (중요)
- 역사/전쟁 주제는 연령 적합하게 순화
- 사실 기반 vs 가상 시나리오 라벨 분리
- 폭력/혐오/자극성 묘사 제한
- 학습 목적(이해/사고/표현) 중심 유지

---

## 9) 데이터 모델 (MVP)
- user_profile(id, grade, level, phonics_mode, preferences)
- story_session(id, user_id, topic, mode, created_at)
- chapter(id, session_id, index, ko_text, en_text, vocab_list)
- question_card(id, chapter_id, q_type, options, correct_or_preferred)
- branch_log(id, session_id, question_id, selected_option)
- opinion_note(id, session_id, chapter_id, lang, text)
- weekly_report(id, user_id, period, study_days, chapters_read, words_count, stress_avg)

---

## 10) KPI (MVP)
1. Day7 재방문율
2. 챕터 완주율
3. 의견 작성률
4. 초입문 모드 사용자 2주 유지율
5. 부모 만족도(주간)

### 임계치(초안)
- Day7 < 35%: 질문 카드/길이 조정
- 완주율 < 60%: 챕터 길이 단축
- 의견 작성률 < 40%: 의견 프롬프트 단순화

---

## 11) 운영/품질 체크리스트
- [ ] 초입문 모드에서 문장 길이 제한 동작 확인
- [ ] 한글/영어 버전 의미 정합성 검수
- [ ] 질문 카드 난이도 과도 여부 점검
- [ ] TTS/재생 속도 옵션 테스트
- [ ] 주간 리포트 누락 여부 점검

---

## 12) 6주 로드맵
- Week 1: 레벨 설정/주제 입력/챕터 생성 v1
- Week 2: 한글/영어 병렬 렌더링 + TTS
- Week 3: 질문 카드 + 1단계 분기
- Week 4: 의견 입력/저장
- Week 5: 주간 리포트 + 부모 화면
- Week 6: 파일럿(3~5가정) + 개선

---

## 13) v3 확정 사항 (사용자 결정 반영)
1. 타깃 학년: **초4~초6**
2. 분기 깊이: **동적 세팅** (기본 1단계, 최대 3단계)
3. 영어 난이도: **다양하게 제공** (L0~L3 자동/수동)
4. 초기 주제: **역사 중심**
5. MVP 1순위 KPI: **Day7 재방문율**

## 14) 추가 확정 사항 (운영 안정화)

### 14-1. 학년별 기본 레벨 매핑
- 초4 기본값: L0~L1 (초입문은 L0.5 우선 권장)
- 초5 기본값: L1~L2 (파닉스 미완성 시 L0.5)
- 초6 기본값: L2~L3
- 부모가 수동으로 상/하향 조정 가능

### 14-2. 동적 난이도 조절 규칙
- 상향 조건: 최근 3세션
  - 정답률 80% 이상
  - 스트레스 평균 2.5 이하
  - 세션 이탈 없음
- 하향 조건: 최근 3세션
  - 정답률 50% 미만 또는
  - 스트레스 평균 4 이상 또는
  - 2회 이상 중도 이탈
- L0.5(Biscuit 수준) 특별 규칙
  - 한 문장 4~7단어
  - 고빈도 sight word 반복 노출 우선
  - 새 단어는 챕터당 3개 이하

### 14-3. 분기 깊이 제어
- 기본값: 1단계
- 몰입 모드: 2단계
- 확장 모드(상급): 3단계
- 보호자 잠금 기능: 1단계 고정 가능

### 14-4. 역사 주제 안전 가이드
- 전쟁/폭력 묘사는 사실 설명 중심, 자극 묘사 금지
- 사실(Fact) / 가정(What-if) 배지를 챕터마다 표시
- 정치적 선동/혐오 표현 차단 필터 적용

### 14-5. 세션 길이 기본값
- 기본: 20분
- 옵션: 15분 / 25분
- 초입문(L0) 기본은 15분 권장

### 14-6. 부모 개입 모드
- Lite: 주 1회 리포트 확인
- Standard(기본): 주 3회 진행 확인 + 주간 코멘트
- Coach: 매일 코멘트 + 다음 세션 난이도 조정

### 14-7. KPI 운영 정의
- MVP 1순위: Day7 재방문율
- 보조 KPI: 챕터 완주율, 의견 작성률
- 목표치(초기):
  - Day7 재방문율 35% 이상
  - 완주율 60% 이상
  - 의견 작성률 40% 이상

### 14-8. 파일럿 과금 정책
- 1차 파일럿(4주): 무료
- 2차 파일럿: 월 구독형 테스트(가벼운 가격)
- 초기 목적: 수익보다 유지율/재방문 검증 우선
