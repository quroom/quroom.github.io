# 오퍼 기준 카테고리 프레임워크 (실수익화 중심)

목적: 콘텐츠 주제가 아니라 **판매 가능한 오퍼 단위**로 카테고리를 정의해,
`콘텐츠 → 리드 → 상담 → 계약` 전환을 만들기.

---

## 왜 이 구조가 필요한가
- 조회수/반응은 높아도 계약으로 이어지지 않으면 수익화가 어렵다.
- 오퍼 기준으로 카테고리를 나누면 각 콘텐츠의 CTA가 명확해진다.
- 무엇을 파는지(문제/결과/가격)가 선명해져 상담 전환율이 올라간다.

---

## 카테고리 정의 (Offer-first)

## 1) AX 자동화 구축 (메인)
### 대상
- 1인 사업자, 소규모 팀, 반복업무가 많은 운영자

### 핵심 문제
- 문의/견적/리포트/정리 등 반복 작업으로 실행력이 떨어짐

### 약속 결과
- 반복 업무 시간 절감
- 누락/실수 감소
- 운영 리듬 안정화

### 대표 오퍼
- 진단(저가): 워크플로우 진단 + 자동화 우선순위 맵
- 구축(중고가): 2주 자동화 스프린트(핵심 1~3개 플로우)
- 유지(구독): 월간 개선 + 장애 대응 + 리포트

### 콘텐츠 연결 CTA 예시
- "원하시면 자동화 점검표 드릴게요. '점검표'라고 남겨주세요."
- "현재 반복업무 3개만 적어주시면 우선순위 제안해드릴게요."

## 2) OpenClaw 셋업/운영
### 대상
- AI 에이전트를 도입하고 싶지만 세팅/운영이 막히는 팀

### 핵심 문제
- 설치는 했는데 운영 규칙, 안전장치, 실제 업무 연결이 안 됨

### 약속 결과
- 실사용 가능한 운영 상태 구축
- 팀 내 재현 가능한 SOP/체크리스트 확보
- 운영 리스크(권한/외부송신/실수) 감소

### 대표 오퍼
- 셋업 패키지: 설치 + 채널 연결 + 기본 자동화 + 가이드
- 운영 온보딩: 팀 교육 + 롤별 사용 시나리오
- 고도화: 워크큐/리포트/하트비트 운영체계 정착

### 콘텐츠 연결 CTA 예시
- "OpenClaw 운영체크리스트 필요하면 '운영표'라고 남겨주세요."
- "현재 막힌 지점 1가지만 주시면 해법 구조를 잡아드릴게요."

---

## 3) Vibe Coding + AI 실행 코칭
### 대상
- 아이디어는 많지만 제품화/배포/운영으로 못 이어지는 솔로 빌더

### 핵심 문제
- 문서만 많고 실제 결과물(배포/고객반응)이 없음

### 약속 결과
- 아이디어를 실행 가능한 시스템으로 전환
- 문서 → 자동화 → 배포 루프 구축
- 4주 내 재현 가능한 개인 실행체계 확보

### 대표 오퍼
- 4주 코칭: 주간 목표 + 실행 피드백 + 산출물 중심
- 템플릿 번들: 바로 쓰는 체크리스트/문서/스크립트
- 동행 구축: 특정 기능/프로세스 대행 구현

### 콘텐츠 연결 CTA 예시
- "실행 루틴 템플릿 필요하면 '루틴'이라고 남겨주세요."
- "현재 멈춘 단계(기획/개발/배포) 적어주시면 다음 액션 제안해드릴게요."

---

## 오퍼 설계 공통 규칙
1. **결과 중심**: 기능 설명보다 결과(시간절감/누락감소/전환증가)를 먼저 제시
2. **범위 명확화**: 해주는 것/안 해주는 것을 문서에 명시
3. **단계 가격**: 저가 진단 → 중고가 구축 → 구독 유지 3단 구조
4. **리스크 문구**: 과장/수익단정 금지, 현실적인 기대치 명시
5. **증거 제시**: 사례, 전후 비교, 체크리스트 스냅샷 제공

---

## 콘텐츠 → 리드 → 상담 → 계약 퍼널
1) 콘텐츠 발행 (문제 공감 + 실행 팁 + CTA)
2) 리드 획득 (댓글 키워드/DM/폼)
3) 15분 진단 콜 (문제/우선순위/범위 정리)
4) 제안서 발송 (범위/일정/비용/산출물)
5) 계약 및 온보딩

핵심 지표:
- 콘텐츠당 리드 수
- 리드→상담 전환율
- 상담→제안서 전환율
- 제안서→계약 전환율
