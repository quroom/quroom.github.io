## 1) PRD

### 1-1. 목표
- 4주 내 첫 워크숍 1회 개최
- 최소 유료 등록 15명 확보
- 강의 후 1:1 유료 상담 5건 전환

### 1-2. 범위
- 랜딩/신청/결제/안내/운영 문서
- 4시간 커리큘럼 + 실습 핸드북
- 후속 전환 퍼널(상담/패키지)

### 1-3. 문제
- AI 자동화에 관심은 높은데 설치/세팅 장벽이 큼
- 툴을 알아도 내 업무에 연결하는 실습 구조가 부족함
- 혼자 설정하다 중도 이탈 비율이 높음

### 1-4. 가설
- G1: “4시간 내 내 업무 자동화 1개 완성” 약속이 등록 전환을 높인다.
- G2: 실습 중심(강의 30%/실습 70%)이 만족도와 재구매를 높인다.
- G3: 강의 종료 직후 48시간 내 후속 제안 시 상담 전환율이 높다.

### 1-5. 비범위
- 장기 부트캠프(8주+) 운영
- 고급 개발자용 심화 커리큘럼
- 법률/투자 자문성 콘텐츠

### 1-6. 리스크/대응
- 모집 저조: 기존 네트워크(LinkedIn/Threads/광주) 우선 모집
- 당일 설치 실패: 사전 체크리스트 + 현장 헬프 2명 확보(협업자/조교)
- 과도한 범위: “완성 1개” 원칙 고정

### 1-7. 지표
- 등록률(랜딩→결제)
- 참석률(결제→실참)
- 수료율(완성 과제 제출)
- NPS/만족도
- 후속 상담 전환률

### 1-8. 체크리스트
- [ ] 오퍼 1문장 확정
- [ ] 커리큘럼 v1 확정
- [ ] 결제/환불 정책 명시
- [ ] 현장 운영표(타임테이블) 확정

### 1-9. 즉시 TODO

2. 신청 페이지 초안 게시
3. 사전 준비물 체크리스트 배포

### 1-10. 검토 질문 3개

## 3) 기능 우선순위 (MoSCoW)

- Must: 랜딩, 결제, 안내 자동화, 사전 세팅 체크
- Should: 후기/사례 자동 수집, 리마인드 자동 발송
- Could: 온디맨드 녹화 판매, 지역별 확장 페이지
- Won’t: 대규모 LMS 구축(이번 사이클 제외)

근거: 첫 사이클은 “모집-운영-후속전환”의 폐루프가 최우선.

## 10) KPI 대시보드 정의

- 랜딩 방문수(일)
- 결제 전환율(주)
- 참석률(회차)
- 수료율(회차)
- 후속 상담 예약률(회차+7일)

임계치
- 결제 전환율 < 3% → 랜딩 카피 수정
- 참석률 < 80% → 리마인드/일정 조정
- 상담 예약률 < 20% → 종료 CTA 재설계
