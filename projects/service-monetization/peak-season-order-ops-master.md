# 피크시즌 주문·재고·픽업 통합 기획서 (통합본)

> 기존 3개 문서(기획 메모 / Living Plan / 어르신용 수량표)를 하나로 합친 **단일 운영 문서**입니다.

## 1) 문제 정의
- 선주문 물량이 현장 판매와 섞여 예약분이 소진됨
- 피크시간 출고/픽업 동선 혼선으로 대기시간 증가
- 시간대별 소진속도 파악이 어려워 추가생산 타이밍 미스
- 노쇼/지연수령 대응 체계 부재로 폐기/CS 증가

## 2) 핵심 컨셉
**예약판매 + 실시간 재고잠금 + 픽업운영 통합**

## 3) 확장 가능한 타깃 업종
1. 떡집/한과점
2. 베이커리/카페
3. 정육/반찬/수산
4. 꽃집
5. 과일/선물세트
6. 델리/밀키트

## 4) 기능 요구사항 (MVP 우선)
### A. 주문/예약
- 예약 주문 생성(품목/수량/수령시간)
- 결제/예약금 상태 관리
- 채널 통합 접수(전화/메신저/현장)

### B. 재고 잠금
- 예약 확정 시 재고 자동 차감
- 예약 재고/현장 판매 재고 분리
- 초과판매(oversell) 차단

### C. 픽업 운영
- 시간대별 픽업 큐
- 주문번호/전화번호로 수령 확인
- 부분수령/대리수령 처리

### D. 운영 대시보드
- 시간대별 주문/출고/픽업 통계
- 품목별 소진 속도
- 예약 잔량/즉시판매 가능 수량

### E. 예외처리
- 노쇼/지연수령 상태 관리
- 환불/변경 이력 기록
- CS 로그 추적

## 5) KPI
- 예약분 누락률 0%
- 초과판매 0건
- 평균 픽업 대기시간 30% 감소
- 노쇼/지연 처리시간 40% 감소

## 6) 2주 MVP 범위
1) 예약주문 등록 + 재고잠금
2) 픽업 확인 화면
3) 시간대별 운영 대시보드(기초)
4) 노쇼/지연 상태 처리

## 7) 어르신용 운영 시트 (현장 즉시 사용)

### 사용 방법(3단계)
1. 주문이 들어오면 주문표에 한 줄 추가
2. 시간대별 합계표 업데이트
3. 합계 기준으로 박스 포장 수량 준비

### 7-3) 절대 규칙
1. 예약분 먼저 분리
2. 예약분 현장판매 혼합 금지
3. 수령 시 이름+시간 확인
4. 전달 후 준비완료 체크
