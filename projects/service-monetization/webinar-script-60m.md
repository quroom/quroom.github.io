# 줌강의 60분 스크립트 / 데모 흐름 / CTA

## 0~5분 오프닝
- 자기소개 (개발자 + 자동화 실무)
- 오늘 목표: "반복업무를 시스템으로 바꾸는 방법"

## 5~15분 문제 공감
- 흔한 문제 3개
  1) 반복작업으로 시간 소모
  2) 누락/실수
  3) 인력 의존도 높음
- before/after 사례 1개 소개

## 15~35분 라이브 데모
- 데모 시나리오:
  - 문의 수집 -> 분류 -> 리포트 자동 생성
- 핵심 포인트:
  - 사람 승인 단계
  - 장애 시 수동 전환
  - 운영 로그 남기기

## 35~45분 참가자 케이스 진단
- 사전 접수한 문제 2개 선택
- 즉석으로 자동화 우선순위 제안

## 45~55분 오퍼 제시
- Starter / Core / Care 소개
- 범위/기간/산출물 명확히 안내
- 선착순 혜택(예: 진단콜 할인)

## 55~60분 CTA 마무리
- CTA1: 진단콜 신청 링크
- CTA2: 자료집 다운로드
- CTA3: 커뮤니티 참여

---

## 진행자 스크립트 샘플 문구
- "오늘은 도구가 아니라 구조를 가져가시면 됩니다."
- "자동화는 기술보다 운영 설계가 먼저입니다."
- "지금 바로 적용 가능한 1개만 정해서 시작하세요."

## 후속 DM 템플릿
"오늘 참여 감사드립니다. 말씀하신 업무 기준으로 30분 진단콜에서 우선순위를 바로 정리해드릴게요. 가능 시간 알려주세요."
