# Service Monetization Execution Playbook

1인 기업 운영 기준으로 BIZ-115~120 문서를 실행 중심으로 통합한 문서.

## BIZ-115 · 고립은둔 완화형 운영원칙 (사회적 연결 중심)

원문: [/projects/service-monetization/social-connection-principles.md](/md-viewer.html?file=/projects/service-monetization/social-connection-principles.md)

## 1) 원칙
1. 성과보다 연결: 매출 목표와 함께 만남/참여 지표를 동시 관리
2. 작은 참여 우선: 댓글 1개, 짧은 질문 1개도 참여로 인정
3. 반복 접점 설계: 1회 이벤트보다 주간 루틴형 만남 운영
4. 심리적 안전감: 모른다고 말해도 괜찮은 분위기 유지

## 2) 운영 가이드
- 무료 오피스아워 주 1회
- 월 1회 오프라인 밋업
- 커뮤니티 신규 참여자 환영 루틴
- 상담 후 7일 이내 후속 연락 1회

## 3) 커뮤니케이션 규칙
- 비교/경쟁 조장 금지
- 과장 성공담보다 실제 시행착오 공유
- 질문 유도형 콘텐츠 우선

## 4) KPI
- 월간 오프라인/온라인 참석자 수
- 신규 참여자 2주 유지율
- 1회 참여 -> 2회 참여 전환율
- 커뮤니티 내 상호작용 수(댓글/질문)

## 5) 리스크 대응
- 참여 저조: 포맷 단순화(30분 짧은 세션)
- 과열/배제 분위기: 운영진 중재 + 가이드 재공지
- 특정 인물 의존: 진행자 분산/게스트 운영

## 6) 실행 체크리스트
- [ ] 주간 오피스아워 일정 고정
- [ ] 월간 밋업 장소/주제 확정
- [ ] 신규 참여자 온보딩 메시지 준비
- [ ] 후속 연락 템플릿 준비

## BIZ-116 · 오프라인 소규모 밋업 파일럿 기획 (전라도권)

원문: [/projects/service-monetization/local-meetup-pilot.md](/md-viewer.html?file=/projects/service-monetization/local-meetup-pilot.md)

## 1) 목적
- 지역 기반 신뢰 형성
- 상담/파일럿 전환 리드 확보
- 고립감 완화형 네트워킹 경험 제공

## 2) 포맷
- 인원: 8~15명
- 시간: 90분
- 구성:
  1) 20분: 자동화 사례 공유
  2) 30분: 참가자 문제 진단
  3) 20분: 소그룹 대화
  4) 20분: 다음 액션 안내(진단콜/커뮤니티)

## 3) 진행 지역(예시)
- 광주 / 전주 / 순천 (접근성 기준)

## 4) 모집 채널
- 네이버카페 공지
- Threads/LinkedIn 공지
- 지인 추천/지역 커뮤니티

## 5) 신청폼 항목
- 이름/직무/업종
- 반복업무 문제 1개
- 밋업 참여 목적
- 후속 상담 희망 여부

## 6) KPI
- 신청자 수
- 참석률
- 만족도(5점)
- 후속 상담 전환율
- 커뮤니티 재참여율

## 7) 실행 체크리스트
- [ ] 장소 확정
- [ ] 발표자료 준비
- [ ] 참가자 안내 메시지 발송
- [ ] 피드백 폼 준비
- [ ] 후속 DM 템플릿 준비

## BIZ-117 · 무료 줌 스터디 / 오피스아워 주간 운영안

원문: [/projects/service-monetization/weekly-office-hours.md](/md-viewer.html?file=/projects/service-monetization/weekly-office-hours.md)

## 목표
- 잠재고객과의 신뢰 형성
- 반복 질문을 공개 세션으로 해결
- 상담콜/패키지 전환 자연화

## 운영 포맷
- 주 1회, 45~60분
- 구성:
  1) 10분: 주간 핵심 인사이트
  2) 25분: 참가자 질문 2~3개 실전 답변
  3) 10분: 다음 액션(진단콜/자료)

## 운영 규칙
- 사전 질문 제출 필수
- 민감한 계정 정보 공유 금지
- 투자권유/법률자문성 질문은 범위 안내 후 종료

## 주간 흐름
- 월요일: 공지 게시 + 신청폼 오픈
- 수요일: 질문 취합/분류
- 금요일: 라이브 진행
- 토요일: 요약/후속 링크 발송

## 전환 포인트
- 세션 종료 시 CTA 2개:
  - 1:1 진단콜 예약
  - Starter 패키지 문의

## KPI
- 신청자 수
- 참석률
- 라이브 후 상담 신청률
- 상담 -> 결제 전환률

## 반복 개선
- 자주 나온 질문은 템플릿 콘텐츠로 전환
- 매월 1회 주제형 특집 세션 운영

## BIZ-118 · 1:1 진단콜 이후 후속 접촉 시나리오 (관계 유지형)

원문: [/projects/service-monetization/followup-retention-flow.md](/md-viewer.html?file=/projects/service-monetization/followup-retention-flow.md)

## 목표
- 상담 후 이탈 감소
- 신뢰 유지
- 유료 전환/재접촉률 향상

## 타임라인
### D+0 (상담 당일)
- 감사 메시지 + 핵심 정리 3줄
- 다음 액션 제안 1개

### D+1
- 맞춤 제안서/요약 자료 전달
- 질문 수집 CTA

### D+3
- 유사 사례 공유
- 적용 가능 체크리스트 전달

### D+7
- 리마인드 메시지
- "이번 주 착수 슬롯" 안내

### D+14
- 미전환 고객 대상 짧은 재진단 제안

## 메시지 템플릿
1) 감사 메시지
"오늘 상담 감사합니다. 말씀주신 핵심 이슈 2가지는 A/B였고, 가장 빠른 개선은 C입니다."

2) 제안서 전달
"요청하신 범위 기준으로 2주 실행안 정리했습니다. 확인 후 우선순위만 확정하면 착수 가능합니다."

3) 리마인드
"지난주 논의한 자동화 안건, 이번 주 착수 가능 슬롯이 있어 안내드립니다."

## 분기 로직
- 반응 있음: 상담 -> 제안 -> 클로징
- 반응 없음: 7일 후 리마인드 -> 14일 후 종료/보류 태깅
- 거절: 사유 기록 -> 30일 후 재접촉

## KPI
- 상담 후 7일 응답률
- 제안서 열람률
- 14일 내 결제전환율
- 30일 재접촉 전환율

## BIZ-119 · 커뮤니티 참여 루프 설계 (신규 → 활동 → 기여)

원문: [/projects/service-monetization/community-engagement-loop.md](/md-viewer.html?file=/projects/service-monetization/community-engagement-loop.md)

## 1) 목적
- 단순 구독자가 아니라 활동/기여 멤버를 늘려
  신뢰 기반 리드와 리퍼럴을 자연 생성.

## 2) 참여 단계
### Stage A: 신규 유입
- 채널: Threads/LinkedIn/카페/무료 줌
- 목표 행동: 가입/신청/첫 댓글

### Stage B: 초기 활동
- 목표 행동: 질문 1개, 사례 반응 1회, 체크리스트 작성
- 지원: 웰컴 가이드 + 첫 액션 템플릿

### Stage C: 반복 참여
- 목표 행동: 주 1회 참여(댓글/질문/경험 공유)
- 지원: 주간 테마 질문, 오피스아워

### Stage D: 기여자 전환
- 목표 행동: 사례 공유, 멤버 도움, 소개/리퍼럴
- 지원: 기여자 배지/소개 혜택/초대권

## 3) 운영 장치
- 웰컴 메시지 자동화
- 주간 체크인 포스트
- 월간 하이라이트(기여자 소개)
- Q&A 라이브(정기)

## 4) KPI
- 신규 -> 첫 활동 전환율
- 주간 활성 참여율
- 재방문율(4주)
- 기여자 비율
- 리퍼럴 발생 수

## 5) 루프 문구 예시
1) "오늘 한 줄 공유: 이번 주 자동화로 줄인 작업 1개"
2) "막힌 지점 질문하면 커뮤니티가 같이 푼다"
3) "내가 받은 도움만큼 한 번은 답한다"

## 6) 리스크/완화
- 소수 독점 대화 -> 진행자가 질문 분산
- 과도한 자기홍보 -> 가이드라인/모더레이션
- 휴면 증가 -> 7일 리마인드/미션형 콘텐츠

## BIZ-120 · 월 1회 하이브리드 데모데이 기획

원문: [/projects/service-monetization/hybrid-demo-day.md](/md-viewer.html?file=/projects/service-monetization/hybrid-demo-day.md)

## 1) 목적
- 오프라인 신뢰 형성 + 온라인 확장 동시 달성
- 잠재고객에게 실제 자동화 결과를 체감시키고 상담 전환

## 2) 운영 형태
- 주기: 매월 1회
- 방식: 오프라인(소규모 10~20명) + Zoom 동시 송출
- 시간: 90분

## 3) 프로그램 구성
1. 15분: 오프닝 + 문제 공감
2. 25분: 실제 자동화 데모 2개
3. 20분: 고객 사례(전/후 지표)
4. 20분: 참가자 Q&A
5. 10분: 진단콜/패키지 CTA

## 4) 준비 체크리스트
- [ ] 장소 예약/장비 점검
- [ ] Zoom 링크/신청폼 생성
- [ ] 발표 자료/데모 시나리오 확정
- [ ] FAQ/가격안/신청 링크 준비
- [ ] 촬영/후기 수집 동의 문구 준비

## 5) 전환 흐름
신청 -> 참석 -> 진단콜 신청 -> 제안서 -> 결제

## 6) KPI
- 신청자 수(오프라인/온라인)
- 참석률
- 행사 후 48시간 내 상담 신청률
- 상담->결제 전환율
- 후기/리퍼럴 발생 수

## 7) 운영 팁
- 지역 특성상 오프라인 네트워크 신뢰를 우선
- 데모는 '기능'보다 '시간 절감' 수치 강조
- 행사 후 24시간 내 후속 메시지 발송

## BIZ-121 · 채널 우선순위 & 마케팅 전략 (수익화 중심)

### 1) 채널 티어링
- Tier 1 (즉시 매출): LinkedIn, 지역 네트워크(광주/전라도), 소개/DM
- Tier 2 (파이프라인 확장): Threads, 랜딩페이지, 오피스아워
- Tier 3 (중장기 자산화): 뉴스레터, 롱폼 콘텐츠

### 2) 오퍼-채널 매핑
- AX 자동화 구축: LinkedIn 사례형 포스트 + 지역 밋업 진단 CTA
- OpenClaw 운영 셋업: Threads 인사이트 + 운영체크리스트 CTA
- Make/n8n 자동화 구축: 랜딩 전환형 콘텐츠 + 15분 진단콜 CTA

### 3) 4주 실행 루프
- W1: ICP 세그먼트 확정 + 채널별 메시지 고정
- W2: 콘텐츠/아웃리치 집행 + 리드 수집
- W3: 상담/제안 집중 + 전환 병목 개선
- W4: 성과 리뷰 + 채널별 재배분

### 4) KPI
- 채널별 리드 수
- 리드→콜 전환율
- 콜→제안 전환율
- 제안→계약 전환율
- CAC 추정(시간비용 포함)

## BIZ-122 · 자동화 운영 전략 (Lead Ops & Compliance)

### 1) 자동화 가능한 업무
- 리드 수집: 공개 출처(회사 웹사이트/디렉토리) 기반 수집
- 리드 정제: 중복 제거, ICP 스코어링, 우선순위 큐
- 아웃리치: 개인화 변수 주입, 발송 스케줄, 후속 시퀀스
- CRM 업데이트: 상태 변경, 다음 액션 리마인드, 주간 리포트

### 2) 권장 파이프라인 (Make/n8n)
1. Source Collector (공개 데이터 수집)
2. Cleaner & Dedup (정제/중복 제거)
3. Scoring (ICP 점수 부여)
4. Outreach Queue (발송 대기열)
5. Follow-up Engine (D+1/D+3/D+7)
6. KPI Reporter (주간 대시보드)

### 3) 컴플라이언스 원칙
- 합법/공개 취득 데이터만 사용
- 대량 스팸성 발송 금지, 소량 테스트 후 확장
- 수신거부(옵트아웃) 경로 제공
- 국가/플랫폼 정책 준수 (개인정보/광고성 정보)

### 4) 운영 리스크
- 발송도메인 평판 악화 → 발송량 제한/워밍업
- 낮은 응답률 → 세그먼트 재정의/오퍼 문구 수정
- 품질 저하 → 수동 QA 샘플링(최소 10%)

## BIZ-123 · 제안서 승률 개선 루프 (거절사유 기반)

### 1) 거절 사유 분류(표준)
- Budget: 예산 미스매치
- Unclear Fit: 적합성 불확실
- Resource: 내부 리소스 부족
- Timing: 우선순위 밀림(나중 검토)

### 2) 사유별 대응 플레이
- Budget → 범위 축소안(Entry) + 단계형 확장안 제시
- Unclear Fit → 14일 파일럿 + 측정 KPI 명시
- Resource → 고객 투입시간 최소 시나리오 제공
- Timing → D+3 재접촉 + 변경된 우선순위 확인

### 3) 제안서 구조 리팩터링 원칙
1. 문제 요약 3줄(고객 언어)
2. 2주 후 결과 3개(숫자)
3. 범위/제외범위 분리
4. 비용 옵션 2안(기본/축소)
5. CTA 1개(다음 행동 명확화)

### 4) KPI
- 제안→계약 전환율
- 거절사유별 비중
- D+3 재접촉 회수율
- 축소안(Entry) 전환율


### 5) 오늘 연락할 Top5 추출 규칙 (Lead Prioritization)
점수식(100점):
- ICP 적합도(0~30)
- 문제 강도/긴급도(0~25)
- 예산 적합성(0~20)
- 최근 반응도(0~15)
- 의사결정 접근성(0~10)

운영 규칙:
1. 매일 아침 리드 풀 점수 재계산
2. 상위 5개만 당일 집중 접촉(Top5 Rule)
3. Top5 외 리드는 자동 후속 시퀀스로 전환
4. 3일 연속 무응답 시 우선순위 하향

태그:
- Hot(80+): 당일 1순위
- Warm(60~79): 48시간 내 접촉
- Cold(<60): 콘텐츠/뉴스레터 육성


## BIZ-124 · 1인기업 일일 실행 루틴 + AI 피드백 루프

## BIZ-125 · 완전 자동화 아키텍처 (Queue Engine + QA Gate + Approval)

### 1) 아키텍처 레이어
1. Intake Layer: 리드/큐 인박스/콘텐츠 아이디어 수집
2. Queue Engine: PENDING→DOING→DONE/BLOCKED 상태기계
3. Execution Worker: 문서작성/리팩터링/링크검증/커밋
4. QA Gate: 금지어/과장표현/중복/링크깨짐 검사
5. Approval Gate: 고위험 변경(외부발송/삭제/민감작업) 사용자 승인
6. Reporting Layer: run-log + daily summary + KPI 대시보드

### 2) 처리 흐름
- Trigger(heartbeat) → Job Pick → Execute → QA → Commit/Push → Log/Report
- QA 실패 시 BLOCKED로 전환 + 입력요청 1~3개 자동 기록

### 3) 안정성 규칙
- WIP 제한: 서비스 2개, 카페 1개
- 타임박스: heartbeat당 최대 20분
- 충돌 방지: 동일 파일 동시작업 금지
- 재시도: 일시 오류는 1회 재시도 후 BLOCKED

### 4) 승인 정책
- 자동 승인: 문서 생성/수정, 내부 링크/템플릿 개선
- 수동 승인 필수: 외부 메시지 발송, 삭제성 변경, 민감정보 포함 가능 작업
