# 상담콜 스크립트 (진단 → 제안 → 클로징)

## 1) 오프닝 (3분)
- 인사 + 목적 확인
- "오늘 목표는 판매가 아니라 우선순위 정리입니다"

## 2) 진단 질문 (10분)
1. 현재 가장 시간이 많이 드는 반복업무는?
2. 누락/실수가 가장 자주 나는 지점은?
3. 자동화되면 가장 먼저 바꾸고 싶은 결과는?
4. 의사결정권자/실행권자는 누구인가?
5. 예산/일정 제약은?

## 3) 문제 정리 (5분)
- 현재 상태 요약
- 핵심 병목 1~2개 확인
- 성공 기준 수치화(시간절감, 누락률)

## 4) 제안 (10분)
- 패키지 추천 (Starter/Core/Care)
- 범위/기간/산출물 명확히 안내
- 제외 범위 및 리스크 투명 안내

## 5) 반론 처리 (7분)
### 예산 부담
- "Starter로 작은 성공부터 시작" 제안

### 내부 인력 없음
- 운영 문서 + 인수인계 포함 강조

### 효과 불확실
- 전/후 지표 측정 방식 제시

## 6) 클로징 (5분)
- 선택지 제시:
  1) 이번 주 착수
  2) 1주 보류 후 재확인
- 다음 액션 확정
  - 제안서 발송 시간
  - 킥오프 가능 일정

## 7) 후속 메시지 템플릿
"오늘 상담 내용을 바탕으로 우선순위 3개와 실행안 정리해 전달드립니다. 확인 후 착수일만 확정하면 바로 시작 가능합니다."


## 후속 메시지 템플릿 (D+0/D+1/D+3)
- D+0: "오늘 상담 감사합니다. 핵심 이슈는 A/B, 가장 빠른 개선은 C입니다."
- D+1: "요청하신 범위 기준 2주 실행안 전달드립니다. 우선순위 1개만 확정해주시면 바로 착수 가능합니다."
- D+3: "유사 사례에서 응답시간 35% 단축된 방식 공유드립니다. 현재 상황에 맞춰 축소안도 가능합니다."


## 아웃리치 메시지 팩 (소개요청/콜드DM/재접촉)
### 1) 소개요청
"요즘 반복업무 자동화 관련해 2주 파일럿을 진행 중인데, 주변에 문의/후속 누락으로 고민하는 대표님 계시면 연결 부탁드려도 될까요?"

### 2) 콜드DM
"안녕하세요, 운영 흐름을 보니 문의→후속 사이 병목이 생길 수 있어 보여서요. 15분 진단으로 우선순위 1개만 잡아드려도 괜찮을까요?"

### 3) 재접촉(D+7)
"지난번 말씀하신 이슈, 이번 주에 바로 적용 가능한 축소안으로 정리해봤습니다. 원하시면 10분만에 공유드릴게요."
