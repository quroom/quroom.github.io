# 04. Offer Scope & Boundary

## In Scope (기본 포함)
- 현재 업무 흐름 인터뷰(60~90분)
- 자동화 우선순위 선정(Top 1~3)
- OpenClaw/Make/n8n 중 적합 스택 제안
- 핵심 플로우 구축 + 테스트
- 운영 가이드(예외처리/담당자/체크포인트)

## Out of Scope (기본 제외)
- ERP/그룹웨어 대규모 커스텀 통합
- 전사 보안/컴플라이언스 컨설팅
- 광고 집행/카피라이팅 대행 자체
- 무제한 변경요청

## 변경관리 규칙
- 범위 추가는 Change Request 문서화 후 일정/비용 재합의
- 긴급 이슈(운영 중단급)는 별도 SLA 기준 적용

## 수락 기준(Definition of Done)
1) 합의한 핵심 플로우가 실제 데이터로 1회 이상 성공 실행
2) 담당자가 가이드 문서만으로 재현 가능
3) 운영 지표 3개가 측정 가능 상태

## Status
- Current: DONE
- Reviewer: user

## Change Request 템플릿 (표준 문구)
- 요청 변경 내용:
- 기존 범위 대비 추가/변경 포인트:
- 일정 영향(일):
- 비용 영향(KRW):
- 승인 필요 항목:

승인 문구:
"본 변경요청으로 인해 일정/비용/산출물 범위가 위와 같이 조정됨에 동의합니다."

## 범위 분쟁 방지 규칙
1) 구두 요청은 효력 없음 (문서 승인 필수)
2) 승인 전 착수 금지
3) 긴급 패치와 기능 확장은 분리 청구
