# 05. Delivery Architecture

## 표준 전달 구조

### Entry (90분 진단)
- 산출물: 문제지도 + 우선순위 + 14일 실행안
- 목적: 결제 전환 가능한 실행 명확화

### Core (2주 스프린트)
- Week 1: 인터뷰/설계/빠른 구현
- Week 2: 테스트/운영적용/문서화
- 산출물: 자동화 플로우 1~3개 + 운영가이드

### Premium (4~8주 구축)
- 단계별 마일스톤 계약
- 핵심 프로세스 다중화 + 운영 대시보드
- 산출물: 팀 운용 가능한 시스템화

## 커뮤니케이션 리듬
- 킥오프 1회
- 주 2회 짧은 싱크(15~20분)
- 주간 리포트 1회(진척/이슈/다음액션)

## 리스크 대응
- 요구사항 변동: 범위 잠금일 지정
- 데이터 접근 지연: 대체 샘플 데이터로 선개발
- 사용자 적응 저항: 운영자 교육 1회 필수
