# 가볍게 스레드 쓰기: 전과정 실행 문서 (기획·개발·운영·마케팅)

## 0) 제품 한 줄 정의
"Threads 창작자가 내 게시글을 빠르게 검색하고, 성과를 분석하며, 메모 기반으로 다음 액션까지 연결하는 유료 툴"

---

## 1) 기획 (Week 1~2)

### 문제 정의
- 게시글 누적 후 재활용/찾기 어려움
- 통계와 실제 원문 연결이 불편
- 아이디어 메모와 성과 데이터가 분리됨

### 핵심 기능(MVP)
1. 내 글 빠른 검색
2. 성과 대시보드 + 원문 바로가기
3. 스친 메모/태그/검색

### 성공 지표
- D7 유지율
- 주간 검색 사용 횟수
- 메모 작성률
- Pro 전환율

---

## 2) 개발 (Week 3~6)

### 기술 스택(권장)
- Front: Next.js
- Back: FastAPI or Django
- DB: Postgres (+ pg_trgm/FTS)
- Queue: Redis/Celery or BullMQ

### 모듈
- Auth/OAuth
- Data Ingestion(API/업로드)
- Post Indexer
- Analytics Aggregator
- Notes Service
- Billing(Stripe)

### MVP 범위
- 계정 연결/업로드 중 1개 경로 먼저
- 검색+필터+정렬
- 성과 리스트→원문 이동
- 메모 CRUD + 태그 + 검색

### 테스트
- 데이터 파서 단위 테스트
- 검색 정확도/성능 테스트
- 권한/삭제 흐름 테스트

---

## 3) 운영 (Week 6~8)

### 운영 대시보드
- 가입/활성/결제/해지
- API 실패율
- 동기화 지연시간

### 고객 지원
- FAQ
- 데이터 연결 가이드
- 장애 공지 템플릿

### 리스크 관리
- 플랫폼 정책 변경 모니터링
- 수집 경로 fallback(업로드 모드)
- 유료 결제 실패 대응 플로우

---

## 4) 마케팅/세일즈 (Week 6~10)

### 타깃
- Threads에서 주 3회 이상 글 쓰는 창작자/1인사업자

### 런칭 플로우
1. Waitlist 오픈
2. 베타 20명 초대
3. 사례 3개 확보
4. Product Hunt/커뮤니티 공개

### 콘텐츠 전략
- "내 글 200개 중 성과 좋은 글 10초 찾기" 데모
- "통계에서 원문 바로 점프" 사용 영상
- "메모→다음 글 소재 전환" 사례

### 가격
- Free / Pro / Team 3단
- 초기 2개월 얼리버드

---

## 5) 로드맵

### Phase 1 (MVP, 6주)
- 검색/통계/메모 핵심 기능

### Phase 2 (Growth, 6주)
- AI 요약/재작성 제안
- 게시글 클러스터링

### Phase 3 (Scale)
- 다채널 확장(X, LinkedIn)
- 팀 협업/권한 관리

---

## 6) 명확한 경계(중요)
- 비공식 우회 수집/보안 회피는 제품 전략에서 제외
- 합법/안전 경로(API/사용자 업로드/동의 기반)만 사용
