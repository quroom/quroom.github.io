# AI 서·논술형 평가 시스템 — Product Engineering Master Plan

## 0) 제품 가설
영어강사/학원/학교는 서술형 평가 채점·피드백·리포트에 많은 시간을 쓰고 있고,
AI 기반 평가 보조 시스템은 **시간 절감 + 피드백 품질 표준화** 가치로 유료화 가능성이 높다.

## 1) 핵심 가치 제안
1. 채점 보조: 루브릭 기반 자동 1차 스코어링
2. 피드백 생성: 수준별 코멘트/수정 제안
3. 리포트 자동화: 학생/학부모/교사용 요약 리포트
4. 이력 관리: 학생별 성장 추적

## 2) 타깃 페르소나
- 영어강사(개인/학원)
- 중소형 어학원 원장
- 학교 방과후/사설 프로그램 운영자

## 3) MVP 범위 (6~8주)
- 학생 답안 업로드(PDF/이미지/텍스트)
- 과제별 루브릭 설정(문법/논리/구성/어휘)
- AI 1차 채점 + 교사 검수 화면
- 피드백 문장 자동 생성
- 결과 CSV/PDF 내보내기

## 4) 기술 아키텍처
- Front: Next.js
- Back: FastAPI(or Django)
- DB: Postgres
- OCR: 한국어/영어 혼합 인식 엔진
- LLM Layer: 채점 프롬프트 + 안전가드
- Audit Log: 점수 산출 근거/프롬프트 버전 기록

## 5) 품질/신뢰 설계 (핵심)
- 사람 검수(HITL) 필수: AI 단독 최종평가 금지
- 루브릭 고정 + 가중치 공개
- 채점근거(trace) 제공
- 오판 사례 수집/재학습 루프

## 6) 운영/컴플라이언스
- 개인정보 최소 수집
- 보관기간/삭제정책 명시
- 미성년자 데이터 처리 동의 절차
- 교육기관 납품 시 보안점검 체크리스트

## 7) 수익모델
- Solo Tutor: 월 구독(학생 수 제한)
- Academy: 팀 요금제(교사 계정/클래스/리포트)
- Institution: 구축형(B2B) + 연 유지보수

## 8) 12주 실행 로드맵
- W1-2: 인터뷰 10명 + 루브릭 샘플 수집
- W3-4: 업로드/루브릭/채점 MVP
- W5-6: 피드백 생성 + 검수 UI
- W7-8: 리포트/내보내기/권한
- W9-10: 베타 5개 반 운영
- W11-12: 과금/세일즈/사례화

## 9) 리스크와 대응
- 평가 공정성 이슈 → 근거 공개 + 교사 최종결정
- 환각/오판 → 금지규칙 + 검수 워크플로
- 도입 저항 → 1주 파일럿 + 시간절감 지표 제시

- [ ] 루브릭 템플릿 3종 작성
