# 리스크/컴플라이언스 체크리스트

## 1) 규제/법적 리스크
- [ ] 투자권유/자문으로 오해될 문구 제거
- [ ] "정보 제공 목적" 고지 문구 상시 노출
- [ ] 성과 보장 표현 금지
- [ ] 면책조항(가정 기반 결과) 명시
- [ ] 필요 시 법률 검토 트리거 정의

## 2) 제품 리스크
- [ ] 입력값 범위 검증(비정상 값 차단)
- [ ] 계산 버전 관리(inputVersion/engineVersion)
- [ ] 재현성 테스트(동일 입력=동일 결과)
- [ ] 시나리오 가정값 문서화
- [ ] 결과 해석 가이드 제공

## 3) 데이터/보안 리스크
- [ ] 개인정보 최소수집
- [ ] 민감정보 암호화 저장
- [ ] 접근권한(Role) 분리
- [ ] 로그에 개인정보 마스킹
- [ ] 백업/복구 정책 수립

## 4) 운영 리스크
- [ ] 장애 대응 프로토콜 (탐지/공지/복구)
- [ ] SLA 목표 설정
- [ ] 배포 전 체크리스트 운영
- [ ] 모델 변경 시 릴리즈 노트 제공
- [ ] 고객 문의 대응 표준 문구 준비

## 5) 비즈니스 리스크
- [ ] 과도한 커스텀 요구 통제
- [ ] 가격/범위/SOW 문서화
- [ ] 환불/해지 정책 명확화
- [ ] 고객 기대관리(가능/불가능 범위)

## 6) 커뮤니케이션 가드레일
권장 문구:
- "이 결과는 가정 기반 시뮬레이션입니다."
- "투자 권유가 아닌 계획 점검 도구입니다."

금지 문구:
- "이대로 하면 은퇴 보장"
- "지금 반드시 매수/매도 필요"

## 7) 런치 전 필수 점검(Go/No-Go)
- [ ] 법적 고지/약관 반영
- [ ] 핵심 시나리오 테스트 통과
- [ ] 보안 점검 완료
- [ ] 고객지원 프로세스 준비
- [ ] 리스크 오너 지정
