BMAD Method 튜토리얼 한국어 가이드

원문 문서 기반 핵심 내용을 한국어로 정리한 요약 번역본입니다.

1) Getting Started

원문: getting-started

  • 가장 먼저 bmad-help를 실행해 현재 상황에 맞는 다음 작업을 추천받습니다.
  • 4단계 흐름: 분석 → 기획 → 솔루션 설계 → 구현
  • 설치: npx bmad-method install
  • 구현 루프: 스토리 생성(SM) → 구현(DEV) → 리뷰(DEV) → 회고(SM)

2) Workflow Map

원문: workflow-map

  • 각 단계 산출물이 다음 단계 입력이 되어 AI 의사결정 일관성을 높입니다.
  • 소규모 작업은 Quick Flow로 1~3단계를 생략하고 빠르게 진행 가능합니다.

3) Agents

원문: agents

  • Analyst, PM, Architect, Scrum Master, Developer, QA 등 역할별 에이전트 제공
  • 각 에이전트는 스킬 ID와 트리거를 통해 호출

4) Skills(Commands)

원문: skills

  • 스킬 직접 호출(bmad-help 등) 또는 에이전트 메뉴 트리거 방식 지원
  • 모듈 변경 시 재설치로 스킬 동기화 필요

5) Official Modules

원문: modules

  • bmb: BMAD 자체 확장(에이전트/워크플로/모듈 제작)
  • cis: 창의적 아이데이션/문제해결 특화
  • gds: 게임 개발 특화
  • tea: 엔터프라이즈 테스트 전략/추적성 특화

6) Testing Options

원문: testing

  • Quinn(기본 QA): 빠른 테스트 생성, 소~중형 프로젝트 적합
  • TEA 모듈: 전략/ATDD/추적성/릴리스 게이트 포함, 대형 프로젝트 적합
